Overview

MeterKenshin is an Android application for reading and managing electricity meters via Bluetooth Low Energy (BLE). It communicates with meters using the DLMS/COSEM protocol, generates billing receipts, and supports printing via Woosim Bluetooth printers.

Key Capabilities

Feature Description
Meter Scan Scans for BLE meters and matches them against a registered meter list
DLMS Reading Reads energy, demand, voltage, and billing data from meters
Billing Calculation Applies rate tables to compute charges automatically
Receipt Printing Prints thermal receipts via Woosim Bluetooth printer
Batch Printing Prints receipts for multiple meters automatically
Data Export Exports billing data as CSV/JSON files
Multi-user Supports Admin and Reader roles with different access levels

Getting Started

Step 1 — Install & Launch

Install the MeterKenshin APK on Android device and open the app.

Step 2 — Log In

Enter username and password on the Login screen. Tap the eye icon to toggle password visibility.

Step 3 — Upload Required Files

Before using the app, upload three required CSV files via Upload Data in the navigation drawer. (meter.csv, rate.csv, printer.csv)

Step 4 — Connect Bluetooth Printer (Optional)

To print receipts, ensure Woosim Bluetooth printer is powered on. The app will auto-connect using the MAC address from printer.csv. (Pair it first with the phone inside bluetooth settings)

Step 5 — Start Reading Meters

Navigate to Meter Reading, select a nearby meter, and tap Read Data in the Meter Detail screen.

Access Description
Home, Meter Reading, Receipt, Settings Standard access
Upload Data Upload and manage CSV files
Hard Reset Reset only personal data
DLMS Rank 3 (read-only access on meter)
Restrictions Cannot access Export Data, cannot edit receipt branding

Required Files

Three CSV files must be uploaded before the app can read meters. Upload them via Navigation → Upload Data.

1. meter.csv — Meter Registry

Defines all meters in the system. The app matches Bluetooth-discovered devices against this file.

Required columns (header row):

Column Type Description
uid Integer Unique row ID
activate 0 or 1 1 = active meter
serialNumber String Meter serial number
bluetoothId String BLE MAC address (e.g., AA:BB:CC:DD:EE:FF)
fixedDate Date Billing schedule date
impKWh Decimal Import energy baseline (kWh)
expKWh Decimal Export energy baseline (kWh)
impMaxDemandKW Decimal Max import demand (kW)
expMaxDemandKW Decimal Max export demand (kW)
minVoltV Decimal Minimum voltage alert threshold (V)
alert Decimal Alert threshold
readDate Date or empty Last reading date (empty = not yet read)
billingPrintDate Date or empty Last print date (empty = not yet printed)
lastCommunication Date or empty Last BLE communication timestamp

Example row:

1,1,12345678,AA:BB:CC:DD:EE:FF,2025-01-15,1234.5,567.8,45.2,12.3,220.5,240.0,,,

2. printer.csv — Printer Configuration

Stores the Woosim Bluetooth printer MAC address for auto-connect.

3. rate.csv — Billing Rate Table

Contains electricity rate values used to calculate billing charges. Must contain 23 rate entries in order (see Billing Calculation).

4. company.csv — Receipt Branding (Optional)

Customizes the company information printed on receipts.

Format (Field, Value pairs):

Field,Value
companyName, Electric Cooperative
addressLine1,123 Main Street Barangay Name
addressLine2,City Province ZIP Code
phone,TEL:02-000-0000
paymentNote,Please pay on or before DUE DATE to avoid disconnection.
disclaimer,This is not an Official Receipt.

This file is optional. If not uploaded, defaults or in-app edits are used.

DLMS Functions

Tap the buttons to perform operations on the connected meter:

Button Description
Registration Authenticate with the meter (establishes DLMS session)
Read Data Read current energy, demand, and billing values
Load Profile Download historical consumption data (supports Get All or Get by Period)
Event Log Download meter event history (supports Get All or Get by Period)
Billing Data Retrieve stored billing records from meter (supports Get All or Get by Period)
Set Clock Synchronize the meter's internal clock

Get by Period (Selective Access)

When tapping Load Profile, Event Log, or Billing Data, a dialog appears with two options:

Option Description
Get All Data Retrieves all available records stored in the meter
Get by Period Retrieves only records within a selected date range (From / To)

When Get by Period is selected, pick the start and end dates using the date picker. The calendar is constrained to the last 12 months, and the "From" date cannot exceed the "To" date (and vice versa). The app sends a DLMS selective access request with a range descriptor to filter records on the meter side, reducing transfer time.

Files saved from a period-based read include the date range in the filename (e.g., 1286_LP_20260318_134707_from20260301_to20260318.csv).

Note: Always do Registration first before any other DLMS operation. If DLMS Confirm Dialog is enabled in Settings, a confirmation dialog appears before each action.

DLMS Log

A scrollable log of all DLMS transactions and responses for the current session. Shows a status chip (Idle / Running / line count), an animated progress bar during operations, and Clear/Stop action buttons. Clear and Stop buttons are hidden during Registration and Read Data operations to prevent confusion.

Saved Billing Data

Shows previously saved billing records for this meter (stored in {serialNumber}_billing.csv).

Print Receipt Button

Sends the latest billing data to the Woosim printer. The button is only active when:

  • The printer is connected
  • Billing data has been read

Workflows

Reading a Meter

1. Ensure meter.csv, printer.csv, and rate.csv are uploaded
2. Navigate to Meter Reading
3. Wait for the meter to appear as "Online" (BLE detected)
4. Tap the meter to open Meter Detail
5. Tap Registration  →  wait for "Session Established" in log
6. Tap Read Data     →  wait for data to populate
7. Billing data is calculated and saved automatically
8. (Optional) Tap Print Receipt if printer is connected

After a successful read, the meter status changes to "Inspected, Billing not Printed" (or Printed if receipt is printed).

Printing a Receipt

Prerequisites:

  • Woosim printer is powered on
  • printer.csv is uploaded with the correct MAC address
  • Billing data has been read for the meter
1. Open Meter Detail for the meter
2. The Bluetooth Printer card shows "Connected"
3. Tap Print Receipt
4. The app sends the receipt to the printer
5. Wait for the receipt to print (~15 seconds max)
6. Meter status updates to "Inspected, Billing Printed"

If the printer is not connected, tap Connect in the Home screen's Bluetooth Printer card, or check that the printer is on and the MAC address in printer.csv is correct.

Batch Printing

Batch printing automatically reads and prints receipts for multiple meters in sequence.

Starting a Batch:

1. Select batch print mode: ALL or NOT_PRINTED_ONLY
   - ALL: print all meters that have billing data
   - NOT_PRINTED_ONLY: skip meters already printed
2. Review the Batch Preview:
   - Total meters to process
   - How many will print vs. be skipped
   - Estimated duration (~7 seconds per meter)
3. Confirm to start the batch

During the Batch:

For each meter, the app will:

  1. Check for existing billing data
  2. Wait for user confirmation (up to 60 seconds)
  3. Send the print job
  4. Wait for print completion (up to 15 seconds)
  5. Move to the next meter

Error Options (if a printer error occurs):

Option Action
Retry Try printing the current meter again (up to 2 auto-retries with backoff)
Skip Skip this meter and continue with the next one
Cancel Batch Stop the entire batch immediately

Resume After Interruption:

Progress is saved after every meter. If the app closes unexpectedly:

  1. Re-open the app
  2. If a saved batch is detected (within 24 hours), a Resume option appears
  3. Tap Resume to continue from where it left off

Batch Summary shows at the end:

  • Total / Success / Failed / Skipped counts
  • Total duration and average time per receipt
  • Success rate
  • Detailed list of any failures with reasons

Exporting Meters (Admin)

This workflow creates a clean meter.csv that Reader users can import.

1. Navigate to Settings
2. Tap Export Registered Meters (Admin Tools section)
3. The app loads all active meters with a billing date assigned
4. A preview dialog shows the list of meters to be exported
5. Tap Export to save to Download/kenshinApp/
   OR tap Share to send via any app (email, messaging, etc.)

What gets exported:

Field Exported value
serialNumber, bluetoothId, uid, activate Unchanged
fixedDate, impKWh, expKWh, demand specs Unchanged
readDate Empty (cleared)
billingPrintDate Empty (cleared)
lastCommunication Empty (cleared)

Readers who import this file will see all meters as "Not Inspected" — ready for a new reading cycle.

Troubleshooting

App doesn't show any meters

  • Check: Is meter.csv uploaded? (Import Data → all 3 files must show green)
  • Check: Is Bluetooth enabled on the device?
  • Check: Are the meters powered on and within Bluetooth range?
  • Check: Does the bluetoothId column in meter.csv match the actual MAC address of the meter?

DLMS Registration fails

  • Ensure do Registration before any other DLMS operation.
  • Check that the DLMS credentials (account, key, logical address) in meter.csv are correct for the target meter.
  • Readers have rank 3 (limited access). Some operations require Admin rank 1.

Printer won't connect

  • Ensure the printer is powered on and in pairing mode.
  • Verify the MAC address in printer.csv matches the printer's Bluetooth address.
  • Go to Settings → Bluetooth Settings → Open App Settings and ensure Bluetooth permissions are granted.
  • Tap Connect in the Bluetooth Printer Status card on the Home screen.
  • If still failing, try turning Bluetooth off and on the device.

Receipt won't print (paper/cover errors)

  • Paper: OUT — Replace the thermal paper roll in the printer.
  • Cover: Open — Close the paper cover on the printer completely until it clicks.
  • After resolving the issue, tap Retry in the error dialog.

App data needs to be reset

  • Current User data only: Settings → Reset User Data (all users see this)
  • Everything (all users): Settings → Hard Reset App (Admin only — irreversible)
